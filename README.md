# ASR5K-Lite CCS Workspace

Public reduced ASR5K-style Code Composer Studio workspace for portfolio and reproducible C2000 examples.

This repository is intentionally separated from the private ASR5K production firmware repository. It is suitable for public demonstration, CCS project-structure reference, and non-confidential F2838x workspace examples.

## Repository Role

- **Purpose:** public, reduced ASR5K demonstration workspace.
- **Status:** public showcase repository.
- **Visibility:** public.
- **Scope:** clean CCS workspace structure, SysConfig examples, and non-confidential project organization.
- **Production status:** not the authoritative ASR5K firmware repository.
- **Production source of truth:** private `ASR5K_V2_Function` repository.

## Current Repository Layout

The repository is kept as a clean CCS workspace with four CCS projects:

```text
ASR5K-Lite/
  .gitignore
  README.md
  sysconfig_cm/
  sysconfig_cpu1/
  sysconfig_cpu2/
  sysconfig_multi/
```

No application source tree is kept at the repository root. CPU-specific source and SysConfig files stay inside their CCS project directories.

## Required TI Components

Minimum tested setup:

```text
C2000Ware        6.00.01.00
SysConfig        1.24.0 or newer
SysConfig tested 1.27.1
C2000 compiler   ti-cgt-c2000_22.6.1.LTS
Device           F2838x / F28388D, 176-pin package
```

C2000Ware 6.00.01.00 requires SysConfig 1.24.0 or newer. If CCS uses an older SysConfig, the build fails before C compilation with:

```text
C2000 SysConfig version 6.00.01.00 requires at least version 1.24.0 of SysConfig.
```

In CCS, confirm product discovery:

```text
Window > Preferences > Code Composer Studio > Products
```

The discovered products should include C2000Ware 6.00.01.00 and SysConfig 1.24.0 or newer.

## Build Verification

The CPU1 and CPU2 RAM configurations can be checked from a shell with:

```powershell
cd .\sysconfig_cpu1\RAM
& 'C:\ti\ccs1281\ccs\utils\bin\gmake' -k -j 16 all -O

cd ..\..\sysconfig_cpu2\RAM
& 'C:\ti\ccs1281\ccs\utils\bin\gmake' -k -j 16 all -O
```

Expected outputs:

```text
sysconfig_cpu1/RAM/empty_sysconfig_cpu1.out
sysconfig_cpu2/RAM/empty_sysconfig_cpu2.out
```

CPU1 and CPU2 FLASH configurations are also expected to build in CCS.

## Project Notes

Device support is generated by SysConfig. The legacy local files below may remain in the project tree but should be excluded from CPU1 and CPU2 builds to avoid duplicate symbols:

```text
device/device.c
device/F2838x_CodeStartBranch.asm
```

The project should not rely on suppressing or filtering compiler/linker problems. Existing `--diag_suppress=10063` is part of the imported CCS project settings; current build fixes should be path/source ownership fixes, not log hiding.

## Common Problems

### `Build-variable COM_TI_C2000WARE_SOFTWARE_PACKAGE_* cannot be resolved`

The project should not depend on deprecated package variables. Refresh or re-import the CCS projects after pulling changes.

### `symbol "Device_init" redefined` or `symbol "code_start" redefined`

The build is compiling both SysConfig-generated device support and legacy local device support files. Confirm `device/device.c` and `device/F2838x_CodeStartBranch.asm` are excluded from CPU1 and CPU2 build configurations.

### `cannot find file ".../driverlib.lib"`

Confirm C2000Ware 6.00.01.00 is installed at the configured path. The tested standard install path is:

```text
C:/ti/c2000/C2000Ware_6_00_01_00
```
